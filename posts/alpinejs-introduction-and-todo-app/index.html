# Alpine.js چیست؟

شاید تا حالا با خیلی از فریم ورک های معروف جاوا اسکریپت آشنا شدین، فریم ورک هایی مثل Vue React Angular و ... که واقعا کدنویسی جاوا اسکریپت رو به سطح جدید آوردن. ولی خب این فریم ورک ها بیشتر واسه پروژه هایی ساخته شدن که تمرکز اصلی یا حداقل تمرکز زیادی روی جاوا اسکریپت باشه. اگر بخوایم یه صفحه ساده html بسازیم و فقط در بعضی از قسمت ها از قابلیت های این فریم ورک ها استفاده کنیم یکم کار سخت میشه و گاهی اوقات حتی به دردسرش هم نمی ارزه.

ولی با Alpine.js قضیه فرق میکنه. آلپاین رو واسه پروژه هایی ساختن که تمرکز اصلی روی جاوا اسکریپت نیست اما خب توی بعضی جاها هم نیازه که از قابلیت های یه فرم ورک جاوا اسکریپت استفاده بشه. مثلا فرض کنین بخوایم یه بخشی از UI رو واکنشگرا یا همون reactive کنیم.

آلپاین رو میشه یجورایی گفت jQuery مدرن و اینطوری کار میکنه که اسکریپتشو به صفحه اضافه میکنیم و تموم، بعدش ازش استفاده میکنیم.

# Alpine رو بهش اضافه می کنیم

alpine.js v3.3.1

توی آلپاین همه چی با دایرکتیو x-data شروع میشه. روی هر عنصر این صفت رو قرار بدیم توسط آلپاین به عنوان کامپوننت شناخته میشه. مقدار این صفت میتونه یه آبجکت شامل داده ها و توابعی باشه که داخل اون کامپوننت قراره استفاده بشه. بهتره که حتما داکیومنت رسمی رو مطالعه کنید، همه چیز کامل توضیح داده شده.

در ادامه با این صفات کار خواهیم کرد من خیلی کوتاه به کاربردشون اشاره میکنم:

  • x-for یه حلقه for که مثلا یه آرایه بهش میدیم و به ازای هر آیتم آرایه، یه عنصر ساخته میشه
  • x-text برای قرار دادن متن در عناصر مختلف
  • x-model برای اتصال داده دو طرفه در input ها (یعنی مقدار متغییر تعریف شده و مقدار input همیشه باهم سینک هستند)
  • x-on یا @ برای اجرای کد در ایونت (event) های مختلف مثلا کلیک
  • x-bind یا : برای قرار دادن مقدار صفت های مختلف

فقط دوتا نکته در رابطه با x-for هست که باید توجه کنید:

  1. همراه x-for باید صفت key هم داده بشه و مقدارش برابر آیدی منحصر به فرد هر آیتم باشه
  2. عنصر داخل x-for نمیتونه خودش کامپوننت باشه و x-data بگیره






 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 

 
 



 
 
 
 
 
 
 
 
 



 


 






<!-- index.html -->

<main>
  <div
    x-data="{
      items: [],
      get checkedItems () {
        return this.items.filter(i => i.checked);
      },
      newItemText: '',
      add () {
        if (!this.newItemText) return;
        this.items.unshift({
          id: Math.random().toString(),
          text: this.newItemText,
          checked: false
        });
        this.newItemText = '';
      },
      remove (index) {
        this.items.splice(index, 1);
      },
    }"
  >
    <input type="text" placeholder="ایتم جدید..." x-model="newItemText" @keydown.enter="add" autofocus>
    <button @click="add" :disabled="!newItemText">
      افزودن
    </button>
    <ul>
      <template x-for="(item, i) in items" :key="item.id">
        <li>
          <label :class="{ 'inactive': item.checked }">
            <input type="checkbox" x-model="item.checked">
            <span x-text="item.text"></span>
          </label>
          <button @click="remove(i)"> × </button>
        </li>
      </template>
    </ul>
    <hr>
    <small>
      تعداد: <span x-text="items.length"></span>
    </small>
    <small>
      تکمیل: <span x-text="checkedItems.length"></span>
    </small>
  </div>
</main>

# لیست رو در localStorage ذخیره میکنیم

خب تا اینجا اپ چک لیست طبق انتظارمون کار میکنه و تنها چیزی که باقی مونده اینه که تغییرات رو در لوکال استورج ذخیره کنیم.

از دو دایرکتیو میتونیم استفاده کنیم:

  • x-init برای خوندن تغییرات از localStorage زمانی که کامپوننت مقدار دهی اولیه میشه
  • x-effect برای ذخیره تغییرات در localStorage زمانی که لیست آپدیت میشه







 
 







<!-- index.html -->

...

<main>
  <div
    ...
    x-init="items = JSON.parse(localStorage.items || '[]')"
    x-effect="localStorage.items = JSON.stringify(items)"
  >
    ...
  </div>
</main>

...

اپ چک لیست ما تکمیل شد. البته این اپ خیلی با نسخه نهایی فاصله داره مخصوصا ظاهرش 😄! ولی خب برای امتحان کردن alpine.js کافی بود.
